# UR10 Robot Driver

> Universal Robots UR10 driver — register-based command protocol over RTDE.
> **File:** `app/robots/ur10.py`
> **Current branch:** `UR`

---

## Connection

| Port | Use |
|---|---|
| `30004` | RTDE — real-time data exchange |
| `29999` | Dashboard — program control |
| `30002` | Secondary — URScript upload |

```python
robot = create_robot("UR10", ip="192.168.X.X")
# Internally:
rtde_receive.RTDEReceiveInterface(ip)
rtde_io.RTDEIOInterface(ip)
```

---

## Protocol Design

Commands sent via **RTDE integer/double registers** — avoids URScript parsing and secondary socket conflicts with ur_rtde daemon.

### Register Map

| Register | Index | Purpose |
|---|---|---|
| `input_int_register_18` | 18 | Command code |
| `input_int_register_19` | 19 | Command ID |
| `input_double_register_18..23` | 18–23 | Target pose or joint angles |
| `input_double_register_24` | 24 | Velocity |
| `input_double_register_25` | 25 | Acceleration |
| `output_int_register_12` | 12 | Robot status |
| `output_int_register_13` | 13 | Ack command ID |

---

## Command Codes

| Constant | Value | Action |
|---|---|---|
| `CMD_IDLE` | 0 | No command |
| `CMD_MOVEL` | 1 | Linear Cartesian move |
| `CMD_MOVEJ` | 2 | Joint move |
| `CMD_GRIP` | 3 | Gripper close |
| `CMD_RELEASE` | 4 | Gripper open |
| `CMD_STOP` | 5 | Stop motion |
| `CMD_FREEDRIVE_START` | 6 | Enable freedrive |
| `CMD_FREEDRIVE_STOP` | 7 | Disable freedrive |

## Status Codes

| Constant | Value | Meaning |
|---|---|---|
| `STATUS_IDLE` | 0 | Ready |
| `STATUS_BUSY` | 1 | Executing |
| `STATUS_DONE` | 2 | Completed |
| `STATUS_ERROR` | 3 | Error |

---

## Mode Constants (RobotBase mapping)

| Constant | Value | Meaning |
|---|---|---|
| `MODE_RUNNING` | `1` | Executing motion |
| `MODE_ERROR` | `9` | Error / safeguard stop (UR robot mode code) |
| `MODE_ENABLED` | `0` | Idle, ready |

> Note: These are UR robot mode codes, not the same as the command server STATUS_* codes. `MODE_ERROR=9` corresponds to UR's protective/safeguard stop state, while `STATUS_ERROR=3` is the URScript command server's error flag.

## URScript Server

`app/robots/ur10_cmd_server.script` — loaded onto the UR controller. It:
- Runs an infinite loop polling RTDE registers
- Executes movel/movej based on CMD register
- Updates status register on completion/error

This script is auto-deployed by the UR10 driver at `__init__` via secondary port 30002.

---

## Init Sequence

The UR10 constructor performs a strict startup sequence:
1. **Probe** — TCP connect to RTDE port (5s timeout, raises `ConnectionError`)
2. **Dashboard** — connect port 29999, send `stop`, wait for `STOPPED/IDLE`
3. **Deploy URScript** — send `ur10_cmd_server.script` via secondary port 30002
4. **RTDE** — connect `RTDEIOInterface` + `RTDEReceiveInterface`
5. **Wait** — poll `output_int_register_12` for `STATUS_IDLE` (up to 15s)

## Unit Conversions

The pipeline uses **mm + degrees (Euler XYZ)**. UR robots use **meters + radians (axis-angle)**. The driver converts transparently:

| Direction | Conversion |
|---|---|
| `move_linear(x,y,z,rx,ry,rz)` | mm→m (`/1000`), Euler deg → axis-angle rad (`_euler_to_axis_angle`) |
| `move_joint_angles(j1..j6)` | deg→rad (`math.radians`) |
| `get_pose()` | m→mm (`*1000`), axis-angle rad → Euler deg |
| `get_angle()` | rad→deg (`math.degrees`) |

## End-Effector — Two Approaches

### 1. Register-based (built-in)
UR10 **overrides** `vacuum_on()`/`vacuum_off()` directly with `CMD_GRIP`/`CMD_RELEASE` register commands. The URScript on the controller handles the gripper action. This **bypasses** the `ToolBase.grasp()/release()` chain.

```python
# In UR10 class — overrides RobotBase.vacuum_on/off:
def vacuum_on(self):
    cmd_id = self._send_command(CMD_GRIP)   # register-based
    self._poll_status(cmd_id, timeout=15.0)

def vacuum_off(self):
    cmd_id = self._send_command(CMD_RELEASE)
    self._poll_status(cmd_id, timeout=15.0)
```

### 2. Dashboard-based (via tool attachment)
Alternatively, [[../Tools/OnRobot RG|OnRobot RG (URCap)]] can be attached as a tool. It uses the UR Dashboard to run saved `.urp` programs. If a tool is attached, the caller would use `tool.grasp()`/`tool.release()` directly, or the register approach is used by default.

```python
from tools.onrobot_urscript import OnRobotRGURScript
tool = OnRobotRGURScript(robot=robot)
robot.attach_tool(tool)
```
